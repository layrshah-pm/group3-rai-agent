# RAI Compliance Agent

**IIMA Capstone · EPAIBBL01 · Group 3**

A LangGraph-based automated AI compliance agent that audits AI-generated outputs in real time, self-corrects violations, and produces a tamper-proof audit trail aligned with the EU AI Act (Article 50) and NIST AI RMF.

---

## Architecture

```
Input → Ingestion → PII Agent → Bias Agent → Policy Agent → Scorecard
                         ↑               ↑            ↑
                         └───────────────┴────────────┘
                                   Correction Node
                              (cycles up to MAX_CORRECTIONS)
```

**Five RAI pillars:** Strategic Alignment · Data Governance · Model Governance · Org Oversight · Continuous Monitoring

## Setup

### Prerequisites

- Python 3.11+
- [Ollama](https://ollama.com/download) with `gemma4:latest` pulled

```bash
ollama pull gemma4:latest
```

### Install dependencies

```bash
pip install -r requirements.txt
python -m spacy download en_core_web_lg
```

### Environment

```bash
cp .env.example .env
# Default settings work if Ollama runs locally on port 11434
```

### Train the demo model (first run only)

```bash
python data/train_model.py
```

This downloads the German Credit dataset and saves `models/loan_model.pkl` with deliberate bias (DI ratio ≈ 0.32) for Scenario 3.

## Running

### Demo (3 scenarios)

```bash
python main.py
```

Expected results:

| Scenario | Expected Status | What it demonstrates |
|---|---|---|
| Clean Compliant Text | `PASS` | All 8 policy criteria satisfied, score 15/15 |
| PII Leakage | `CORRECTED` or `ESCALATED` | Presidio detects + redacts PII; policy loop fires on bare text |
| Biased Model Output | `ESCALATED` | Fairlearn DI ratio = 0.316; LLM cannot fix model weights |

### Streamlit UI

```bash
streamlit run ui/app.py
```

Opens at `http://localhost:8501`. Two tabs:
- **Text Auditor** — paste text, run live audit with streaming node progress and radar chart
- **Model Auditor** — upload CSV, select protected attribute, view fairness metrics

## Demo scenarios

### Scenario 1 — Clean Compliant Text

Input satisfies all 8 regulatory criteria:
- **TRANSPARENCY** — "generated by an AI system"
- **EXPLAINABILITY** — three decision factors listed explicitly
- **HUMAN_OVERSIGHT** — "request human review at any time"
- **DATA_QUALITY** — names data source + acknowledges limitation
- **NON_DISCRIMINATION** — neutral language, no demographic use
- **PRIVACY** — no personal identifiers
- **ACCOUNTABILITY** — "First National Bank" named as responsible party
- **ROBUSTNESS** — confidence score 0.87 stated

Expected: `PASS`, overall score 15/15.

### Scenario 2 — PII Leakage

Input contains: `PERSON` (John Smith), `EMAIL_ADDRESS`, `PHONE_NUMBER`.

Presidio detects and redacts in one correction cycle. Post-redaction, the bare text
lacks policy criteria (no AI disclosure, no explanation) → policy agent fires →
loop escalates after 3 attempts.

Expected: `ESCALATED` (PII corrected; underlying policy non-compliance persists).

### Scenario 3 — Biased Model Output

Input: `input_type="model_output"` with `gender="F"`, `protected_attributes=["sex"]`.

Pre-trained model has deliberate bias (female+bad samples upweighted 4×):
- Disparate Impact ratio: **0.316** (threshold: ≥ 0.8)
- Demographic Parity diff: **0.413** (threshold: ± 0.1)
- Equalized Odds diff: **0.478** (threshold: ± 0.1)

LLM correction can reword the output text but cannot fix the underlying model weights →
escalates after 3 attempts.

Expected: `ESCALATED`.

## Querying the audit database

`rai_audit.db` is created automatically on first run.

```python
from utils.audit_logger import get_audit_trail, get_runs_summary, get_node_timings

# All past runs
for run in get_runs_summary():
    print(run["thread_id"][:8], run["final_status"], run["total_score"])

# Full audit log for a specific run
entries = get_audit_trail(thread_id)
for e in entries:
    print(e["timestamp"], e["node"], e["action"], e["result"])

# Node execution times
timings = get_node_timings(thread_id)
```

Direct SQLite access:
```bash
sqlite3 rai_audit.db "SELECT thread_id, checkpoint_id FROM checkpoints ORDER BY rowid DESC LIMIT 10;"
```

## RAI pillar scores

| Pillar | Score Key | Primary Agent |
|---|---|---|
| Strategic Alignment | `strategic_alignment` | Policy Agent |
| Data Governance | `data_governance` | PII Agent + Bias Agent |
| Model Governance | `model_governance` | Bias Agent |
| Org Oversight | `org_oversight` | Policy Agent |
| Continuous Monitoring | `continuous_monitoring` | Correction history |

Scores: 0 = non-compliant · 1 = corrected · 2 = minor gaps · 3 = fully compliant · Max = 15
